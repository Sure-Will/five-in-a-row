# five-in-a-row

一个单文件的五子棋前端项目，主打赛博风界面、响应式布局和轻量 AI 对手。

[English](./README.md) | [简体中文](./README.zh-CN.md)

## 项目简介

这个项目本质上是一个纯静态浏览器游戏，核心文件只有一个：[`index.html`](./index.html)。它的目标不是复杂工程化，而是：

- 打开就能跑
- 部署门槛低
- 前端结构清楚，方便继续改

当前版本重点放在：

- 单页 UI 体验
- 桌面端和手机端都能直接操作
- AI 先手天元，并用威胁扫描加轻量 Alpha-Beta 做应手
- 本地最佳通关时间记录
- 不依赖构建工具和第三方运行时

## 当前玩法

- AI 执白先手。
- 玩家执黑后手。
- 点击或触摸棋盘交叉点即可落子。
- 任意方向先连成五子的一方获胜。
- 对局过程中会同步更新状态面板、当前回合、步数、延迟和消息流。

## 当前特性

- 单文件实现：结构、样式、逻辑都放在 `index.html`
- 响应式布局：兼容桌面、平板和手机
- 触控支持：手机上可以直接点按棋盘落子
- 实时状态反馈：系统状态、当前回合、计时、步数、消息面板
- 本地记录保存：最佳胜利用时写入浏览器 `localStorage`
- 静态部署友好：适合 GitHub Pages 或任意静态托管

## 本地运行

你可以直接打开 HTML 文件，但更推荐起一个本地静态服务来验证页面行为。

### 方式一：直接打开文件

直接用浏览器打开 [`index.html`](./index.html)。

### 方式二：本地起服务

```bash
cd five-in-a-row
python3 -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

## 手机端验证

如果你想重点检查手机端适配，建议这样测：

1. 先起本地服务。
2. 在浏览器里打开页面。
3. 用开发者工具切到手机尺寸，或者直接用同一局域网下的手机访问。
4. 重点看这几项：
   - 页面有没有横向溢出
   - 棋盘会不会跟着小屏缩放
   - 手指点交叉点能不能稳定落子
   - 按钮、状态面板、日志文字在小屏上是否还能读

## 实现说明

- 棋盘渲染基于 HTML5 Canvas
- UI 使用原生 HTML + CSS
- 游戏逻辑使用原生 JavaScript
- 当前 AI 属于启发式轻量实现，重点是可玩和前端交互完整

它不是一个比赛级引擎，更像一个完成度不错、便于继续迭代的静态前端五子棋项目。
